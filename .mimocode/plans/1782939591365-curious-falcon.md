# Plano: TradNinja Studio - Tradutor Offline do Zero

## Onde
`C:\Users\clept\OneDrive\Documents\academia\tradninja-studio`

## O que é
Um tradutor offline completo, separado do projeto tradninja atual. Usa Argos Translate (modelos neurais) para traduzir PT→EN/ES/FR sem internet após download dos modelos.

## Arquitetura

```
tradninja-studio/
├── package.json
├── tsconfig.json
├── src/
│   ├── index.ts              # Export principal
│   ├── engine.ts             # Motor de tradução (usa Python por baixo)
│   ├── dictionary.ts         # Dicionário curado (cache local)
│   ├── types.ts              # Tipos TypeScript
│   └── cli.ts                # CLI para traduzir arquivos
├── models/                   # Modelos Argos Translate (baixados 1x)
│   ├── pt_en.argosmodel
│   ├── pt_es.argosmodel
│   └── pt_fr.argosmodel
├── data/
│   ├── pt.json               # Fonte (palavras PT)
│   ├── en.json               # Traduções EN (geradas)
│   ├── es.json               # Traduções ES (geradas)
│   └── fr.json               # Traduções FR (geradas)
├── scripts/
│   ├── setup.py              # Instala Argos Translate + modelos
│   ├── translate.py          # Script de tradução offline
│   └── verify.py             # Verifica qualidade das traduções
└── README.md
```

## Passo 1: Setup inicial
- Criar pasta `tradninja-studio`
- `npm init` com TypeScript
- Instalar dependências Node: typescript, tsx

## Passo 2: Script Python de tradução (`scripts/translate.py`)
- Instala argostranslate automaticamente se não tiver
- Baixa pacotes pt→en, pt→es, pt→fr
- Lê pt.json, traduz cada palavra, salva nos JSONs de destino
- Salva progresso incremental (retomável)
- ~2-3 minutos para 15.000 × 3 idiomas

## Passo 3: Motor TypeScript (`src/engine.ts`)
- Função `translate(text, from, to)` que chama o Python
- Função `translateBatch(words, from, to)` para múltiplas palavras
- Cache local em JSON para não re-traduzir
- Tipagem completa

## Passo 4: CLI (`src/cli.ts`)
- `npx tsx src/cli.ts translate --from pt --to en --input data/pt.json --output data/en.json`
- `npx tsx src/cli.ts translate --word "casa" --from pt --to en`
- `npx tsx src/cli.ts verify --lang en` (mostra amostras)

## Passo 5: Integração com tradninja
- O tradninja atual importa de tradninja-studio
- Substitui os JSONs lixo por traduções reais

## Execução
1. Criar pasta e configurar projeto
2. `python scripts/translate.py` (~3 min)
3. Verificar traduções
4. Usar como biblioteca ou CLI
